# 车载低轨卫星通信系统EMC正向设计方案（双轨合并版）

## 1. 编制定位

本稿不是把两份现有规范简单拼接，而是采用“共性主文 + 双轨分层”的方式重组，保证两份来源文件在合并后仍然各自成体系。

- `A轨`：一汽-工创方案线  
  以 `正向设计规范0312.docx` 为主，保留一汽工程指标、工创中心硬件集成与抗干扰思路、仿真平台与项目化落地要求。
- `B轨`：专利-标准增强线  
  以 `Q_FC_XXX-2025_车载低轨卫星通信系统EMC性能正向设计规范.md` 为主，保留专利映射、行业标准限值、测试判据与规范化条款表达。
- `共性层`：两轨共同认可且不存在冲突的内容  
  作为主文正文，直接形成统一的正向设计方案骨架。

## 2. 合并原则

### 2.1 双轨一体，不做硬揉平

- `共性层` 只保留两份规范都支持的设计逻辑、基本条款和总流程。
- `A轨` 保留项目指标、工程实现细节、器件与平台选型、仿真链路与实车目标。
- `B轨` 保留可追溯的标准条款、专利支撑、试验限值、性能判据和附录化表达。

### 2.2 冲突处理规则

- 若 `A轨` 是更严或更可落地的工程指标，内部研发阶段优先采用 `A轨`。
- 若 `B轨` 是标准明确限值或测试判据，对外发布、评审、立项和验收阶段优先采用 `B轨`。
- 若两者不冲突，则正文写 `共性要求`，`A轨/B轨` 分别放入对应子条款或资料性附录。

### 2.3 推荐成文方式

- 正文采用三段式写法：`共性要求`、`A轨实施要求`、`B轨增强要求`。
- 不再把 `A轨` 和 `B轨` 分成两份完全独立的全文，而是在每一设计层内保持并列。
- 对外若需要标准稿，可直接抽取 `共性层 + B轨`。
- 对内若需要工程实施稿，可直接抽取 `共性层 + A轨`。

## 3. 统一后的总体目标

### 3.1 共性目标

- 在设计初期完成“预测-规避-抑制-验证”的闭环，而不是依赖后期整改。
- 覆盖系统架构、天线与射频前端、硬件平台、线束布局、测试验证五个层面。
- 满足车载低轨卫星通信系统在复杂车载电磁环境中的发射合规、抗扰可用和业务连续。

### 3.2 A轨工程目标

`A轨` 建议继续保留以下工程指标，作为项目化实施目标：

| 指标项 | 建议目标 |
|---|---|
| 数据通信频率 | 上行 1.6 GHz / 下行 1.5 GHz |
| 语音业务速率 | ≥ 2.4 kbps |
| 接收灵敏度 | 优于 -120 dBm |
| 宽带通信速率 | > 20 Mbps |
| Ku波段 EIRP（含天线罩） | ≥ 1 dBw（仰角 ≥ 10°） |
| Ku波段 G/T（含天线罩） | ≥ -27.5 dB/K（仰角 ≥ 10°） |
| 路测通信信噪比 | ≥ 1 dB |
| 路测信号强度 | ≥ 1 dBm |
| 抗多普勒频率变化率 | 300 Hz/s |

### 3.3 B轨合规目标

`B轨` 建议继续保留以下合规目标，作为标准化与验收目标：

- 满足 `YD/T 4517.4-2024` 对辐射杂散、辐射骚扰、传导骚扰、ESD、辐射抗扰度、BCI、电瞬态的要求。
- 保持 `性能判据A/B/C` 的完整表达和试验适用边界。
- 保持专利与条款的可追溯关系，便于立项说明、标准评审和知识产权风险识别。

## 4. 统一后的总技术路线

### 4.1 主线流程

1. 识别系统共存对象与干扰路径。
2. 在系统架构层先做规避，再在天线、硬件、线束层做抑制。
3. 用仿真提前暴露最不利工况。
4. 用标准化测试完成验收闭环。

### 4.2 双轨分工

| 层级 | 共性层角色 | A轨角色 | B轨角色 |
|---|---|---|---|
| 系统架构 | 定义共存与调度骨架 | 落到项目工作模式与实车场景 | 落到专利与标准条款 |
| 天线射频 | 定义方向性、滤波、布局 | 落到性能指标、算法响应、器件参数 | 落到辐射限值与专利映射 |
| 硬件平台 | 定义屏蔽、接地、电源框架 | 落到屏蔽罩、材料、效率、输入范围 | 落到抗扰度和瞬态试验要求 |
| 线束布局 | 定义布线与搭铁原则 | 落到间距、绞距、端接方式 | 落到BCI与瞬态抗扰试验边界 |
| 测试验证 | 定义验证闭环 | 落到仿真平台与模型库 | 落到标准方法、限值和判据 |

## 5. 系统架构层合并方案

### 5.1 共性要求

- 在设计初期识别卫星通信、GNSS、4G/5G、Wi-Fi、蓝牙等共存模块之间的频域、时域与空间干扰关系。
- 建立统一的资源调度和通信状态管理机制，避免多模块无序并发。
- 将时域隔离、网络切换、多通道管理和免测频段规划写入系统级设计要求。

### 5.2 A轨实施要求

- 将自干扰分析前移到方案阶段，明确最不利工况而不是仅在样机阶段排查。
- 当卫星发射抑制 GNSS 时，应保留惯性导航等备份手段，保证基本定位能力连续。
- 结合实车使用场景，对高速行驶、多普勒变化和网络切换时延做联合分析。
- 在仿真阶段直接评估卫星通信模块、定位模块、蜂窝模块共存下的业务连续性。

### 5.3 B轨增强要求

- 采用专利化表达保留时域隔离机制、卫星/蜂窝切换逻辑和多数据通道选择逻辑。
- 明确发信机免测频段和性能判据A的标准边界，避免条款口径模糊。
- 继续保留对 `CN202410892217.1`、`CN202411493441.X`、`CN202311559261.2` 的映射关系。

### 5.4 合并写法建议

- 正文写“系统级共存设计要求”。
- `A轨` 子条款写“实车场景与业务连续性保障”。
- `B轨` 子条款写“条款来源与合规边界”。

## 6. 天线与射频前端层合并方案

### 6.1 共性要求

- 采用多阵元平面阵列天线或同等空间选择性方案。
- 天线优先布置于车顶，主辐射方向朝向外部空域。
- 射频链路应配置滤波、双工和必要的带外抑制措施。
- 天线与射频前端需同时考虑发射约束与接收保护。

### 6.2 A轨实施要求

- 多天线之间隔离度建议按自干扰分析结果控制，工程上宜不低于 `30 dB`。
- 保留 Ku 波段 `EIRP`、`G/T`、路测 `SNR`、信号强度等项目指标。
- 波束赋形算法应兼顾实时性与抗干扰能力，响应时间建议 `≤ 100 ms`。
- 对已知干扰方向，零陷深度建议 `≥ 20 dB`，自适应收敛时间建议 `≤ 10 ms`。
- 发射链路滤波器可继续采用“插入损耗 `≤ 1.5 dB`、带外抑制 `≥ 40 dB @ ±2倍信道带宽`”的工程要求。
- 接收链路建议保留 `NF ≤ 2 dB` 的目标。
- 天线馈线、控制线的屏蔽和 360° 端接要求应继续保留。

### 6.3 B轨增强要求

- 保留相邻天线单元在 `-5 dB` 处重叠、合成波束覆盖 `30°～150°` 仰角范围的条款表达。
- 保留辐射杂散 `30 MHz～1 GHz：-36 dBm`、`>1 GHz：-30 dBm` 的标准限值口径。
- 保留辐射抗扰度 `400 MHz～6 GHz`、`90%频段 30 V/m` 的标准要求。
- 保留天线角度自适应调整、二分法求解、下线检测等专利映射条款。

### 6.4 合并写法建议

- 正文写“天线系统与射频链路的一般设计要求”。
- `A轨` 写“工程性能与实现参数”。
- `B轨` 写“标准限值与专利条款增强要求”。

## 7. 硬件平台层合并方案

### 7.1 共性要求

- 核心模块应进行金属屏蔽、分区布局、良好接地和电源净化。
- 数字、射频、电源三个区域应清晰分离。
- 电源端口和信号端口都应考虑瞬态防护。
- 外部连接器应满足机械可靠性与接口屏蔽的双重要求。

### 7.2 A轨实施要求

- 继续保留核心模块独立屏蔽罩方案，可覆盖主控、低轨通信模块、Ku终端等关键单元。
- 屏蔽罩材料、厚度、屏蔽效能、开孔方式等要求宜作为工程实施参数保留：
  - 材料可采用镀锡钢或洋白铜；
  - 厚度建议 `≥ 0.3 mm`；
  - 屏蔽效能建议 `30 MHz～6 GHz ≥ 40 dB`；
  - 开孔孔径建议 `≤ 3 mm`。
- 接地设计建议保留“低频单点、高频多点”的混合接地方案。
- 接地电阻、磁珠/0欧连接、电源输入 `EMI` 滤波、`9V～36V` 宽压输入、`≥ 85%` 电源效率等工程条款建议保留在 `A轨`。
- 功耗模式切换和模块级上电策略也适合留在 `A轨`。

### 7.3 B轨增强要求

- 保留 `λ/20` 缝隙处理原则。
- 保留 ESD、电源线瞬态、非电源线瞬态、传导/辐射骚扰等标准试验边界。
- 保留二次锁扣快速插拔接口的专利支撑与规范化描述。

### 7.4 合并写法建议

- 正文写“硬件平台 EMC 结构要求”。
- `A轨` 以“屏蔽、接地、电源、功耗”的工程条款展开。
- `B轨` 以“抗扰度、骚扰和接口规范化要求”展开。

## 8. 线束布局层合并方案

### 8.1 共性要求

- 敏感线束优先采用屏蔽或双绞结构。
- 布线遵循“源-路径-敏感体”分离原则。
- 环路面积尽量最小化，回流路径尽量完整。
- 线束屏蔽、搭铁和 BCI 防护需统筹设计。

### 8.2 A轨实施要求

- 所有敏感信号线可继续按屏蔽线缆控制，屏蔽层采用 `360°` 端接，避免“猪尾巴”接地。
- 差分信号线建议保留双绞线工程约束，例如 `100 Mbps` 以太网绞距 `≤ 50 mm`。
- 强干扰线束与敏感线束的工程间距建议保留，宜 `≥ 5 cm`。
- 线束交叉宜垂直，必要时可通过磁环、共模扼流圈补强。
- 设备搭铁点选取、线束两端接地与地环路风险控制更适合放在 `A轨`。

### 8.3 B轨增强要求

- 保留屏蔽线缆、双绞线、环路面积最小化等规范化表达。
- 保留 `GB/T 21437.3` 和 `YD/T 4517.4-2024` 对非电源线瞬态抗扰度的表述。
- 保留 `BCI` 试验要求，即 `1 MHz～400 MHz` 频段 `90%` 以上频点注入电流 `60 mA`。

### 8.4 合并写法建议

- 正文以布线方法论为主。
- `A轨` 给出间距、绞距、端接方式等工程值。
- `B轨` 给出试验判据和表格化口径。

## 9. 测试验证层合并方案

### 9.1 共性要求

- 验证必须覆盖仿真、样件测试、实车或实验室对标三个阶段。
- 不只验证“是否通过”，还要验证“问题出现在哪里、为何出现、如何回灌设计”。
- 验证应同时覆盖骚扰发射、抗扰度和业务性能。

### 9.2 A轨实施要求

- 继续保留 `EMC` 仿真平台要求，包括通信模组电磁模型、车辆典型电磁环境模型、多普勒模型、联合仿真环境和模型校准机制。
- 保留“设计初期以仿真主导、样件阶段以测试为最终判据”的协同逻辑。
- 建议继续沉淀干扰源模型库、天线模型库和典型工况库，服务后续复用。

### 9.3 B轨增强要求

- 保留 `YD/T 1483`、`GB/T 18655-2018`、`GB/T 19951`、`GB/T 33014.2`、`GB/T 33014.4`、`GB/T 21437.2`、`GB/T 21437.3` 等方法学引用关系。
- 保留 `性能判据A/B/C` 原始适用场景。
- 保留辐射杂散、辐射骚扰、传导骚扰、ESD、辐射抗扰度、BCI、电瞬态的表格化测试要求。

### 9.4 合并写法建议

- 正文拆成 `仿真验证` 和 `合规验证` 两节。
- `A轨` 放仿真平台、模型和工程验证闭环。
- `B轨` 放标准方法、限值、判据和附录化清单。

## 10. 推荐交付体例

### 10.1 主文结构

- 第1章至第4章写 `共性层`。
- 第5章至第9章每章内固定使用 `共性要求 / A轨实施要求 / B轨增强要求 / 合并写法建议` 四段式。
- 不再另起一份完全重复的“基础版”或“增强版”全文。

### 10.2 配套附件

- `附件A`：一汽-工创实施条款汇编  
  只抽取 `A轨`，便于研发、器件选型、平台搭建和项目管理。
- `附件B`：专利与标准条款映射表  
  只抽取 `B轨`，便于评审、对外说明和知识产权审查。
- `附件C`：统一检查清单  
  只保留最终落地必须检查的项目，避免两套清单并存。

## 11. 使用建议

- `内部研发版`：采用 `共性层 + A轨`，`B轨` 作为验收和对标依据。
- `标准评审版`：采用 `共性层 + B轨`，`A轨` 下沉为资料性说明或实施导则。
- `项目汇报版`：采用 `共性层 + A轨核心指标 + B轨关键判据`，避免全文过重。

## 12. 最终建议

这次合并不建议继续沿用“一个完整版把所有内容写在同一层级”的方式，而应固定为：

- 一个主文，承载共性骨架；
- 一条 `A轨`，承载一汽-工创工程落地；
- 一条 `B轨`，承载专利-标准增强与合规；
- 一套统一检查清单，承载最终执行。

这样处理后，两份来源规范在逻辑上是合并的，在使用上是独立的，在后续维护上也更容易分开迭代。
